## 모두의 네트워크 교재 정리 (약 5문제 출제)

### 1장 : 네트워크 첫걸음

- `네트워크` : (두 대 이상의) 컴퓨터 간의 연결. 네트워크를 통해 데이터를 서로 주고받음
- `인터넷` : 전 세계의 네트워크를 연결한 거대한 네트워크
- `대역폭(bandwidth)` : 단위 시간당 전송량
- 디지털 데이터 : 0과 1의 집합
- 비트와 바이트 (8비트 = 1바이트)
- `LAN` : 건물 안이나 특정 지역을 범위로 하는 네트워크
- `WAN` : 지리적으로 넓은 범위에 구축된 네트워크
- ISP(인터넷 서비스 제공자)가 제공하는 서비스를 사용하여 LAN과 LAN을 연결하는 WAN 네트워크를 구축할 수 있다.
- LAN은 WAN에 비해 오류가 발생할 가능성이 낮다. LAN이 WAN보다 속도가 빠르다.
- 가정에서 랜을 구성하려면 `인터넷 서비스 제공자`와 `인터넷 회선`을 결정하고 계약한다.
- `DMZ` : 외부에 공개하기 위한 네트워크
- `온프레미스` : 사내 또는 데이터 센터에 서버를 두고 운영하는 것

### 2장 : 네트워크의 기본 규칙

- `프로토콜` : 통신하기 위한 규칙
- `OSI 7계층` 및 `TCP/IP 4계층`
- `헤더` : 전송되는 데이터 맨 앞에 위치하는 추가적인 정보 데이터
- `캡슐화` : 헤더를 붙여 나가는 과정
- `역캡슐화` : 헤더를 하나씩 제거해나가는 과정
- `트레일러` : 데이터를 전달할 때 데이터의 마지막에 추가되는 정보
- `VPN` : 가상 통신 터널을 만들어 기업 본사나 지사와 같은 거점을 연결하여 통신하거나, 외부에서 인터넷으로 사내에 접속하는 것

### 3장 : 물리계층

- `물리 계층` : 데이터를 전기 신호로 변환하는 계층
- `랜 카드` : 데이터를 전기 신호로 변환하는 장치
- `전송 매체` : 데이터가 흐르는 물리적 선로
- `트위스트 페어 케이블` : UTP 케이블, STP 케이블이 있다
- `UTP 케이블` : 실드로 보호되지 않은 케이블 (노이즈의 영향을 받는다)
- `STP 케이블` : 실드로 보호한 케이블
- UTP 케이블의 분류

| 분류  | 규격        | 속도     |
| ----- | ----------- | -------- |
| Cat3  | 10BASE-T    | 10Mbps   |
| Cat5  | 100BASE-TX  | 100Mbps  |
| Cat5e | 1000BASE-T  | 1000Mbps |
| Cat6  | 1000BASE-TX | 1000Mbps |
| Cat6a | 10GBASE-T   | 10GMbps  |
| Cat7  | 10GBASE-T   | 10GMbps  |

- 랜 케이블의 양 끝에는 `RJ-45` 커넥터가 있음
- 다이렉트 케이블(다른 장치 간) / 크로스 케이블(같은 장치 간)
- `리피터` : 전기 신호를 정형/증폭하는 장비
- `허브(더미 허브)` : 컴퓨터 여러 개 연결 + 리피터 기능 (수신한 데이터를 모든 포트로 전송)

### 4장 : 데이터링크 계층

- `데이터링크 계층` : 네트워크 장비 간 신호를 주고받는 규칙 정하는 계층
- `이더넷` : 여러 컴퓨터가 동시에 데이터 전송해도 충돌이 일어나지 않는 규칙 (CSMA/CD)
- `MAC 주소` : 물리 주소. 전 세계에서 유일한 번호 (ex. 00-23-AE-D9-7A-9A)
- `이더넷 헤더` : 목적지 MAC주소(6바이트) + 출발지 MAC주소(6바이트) + 유형(프로토콜 종류 식별하는 번호)(2바이트) = 총 14바이트
- `프레임` : 이더넷 헤더와 트레일러가 추가된 데이터
- 허브에서 데이터를 모든 포트로 무지성 전송하면 -> MAC주소로 자신의 데이터인지 비교
- `스위치` : MAC 주소 테이블을 가짐. 똑똑한 허브! 데이터 링크 계층에서 동작함
- `MAC 주소 테이블` : 스위치의 포트 번호와 해당 컴퓨터의 MAC 주소가 등록된 DB
- `MAC 주소 학습 기능` : MAC 주소가 등록되어 있지 않으면 포트와 함께 등록하는 스위치의 기능
- `플러딩` : 등록되어있지 않으면 등록되지 않은 모든 포트에 데이터를 전송하는 것
- `MAC 주소 필터링` : MAC 주소를 기준으로 목적지를 선택하는 것
- 허브는 `반이중 통신 방식`, 스위치는 `전이중 통신 방식` 사용
- `충돌 도메인` : 충돌이 발생할 때 그 영향이 미치는 범위 (허브는 모든 컴퓨터가 충돌 도메인, 스위치는 컴퓨터 하나하나)
- 이더넷 규격

| 규격 이름  | 통신 속도 | 케이블                | 케이블 최대 길이 | 표준화 연도 |
| ---------- | --------- | --------------------- | ---------------- | ----------- |
| 10BASE5    | 10Mbps    | 동축케이블            | 500m             | 1982년      |
| 10BASE2    | 10Mbps    | 동축케이블            | 185m             | 1988년      |
| 10BASE-T   | 10Mbps    | UTP케이블 (Cat3 이상) | 100m             | 1990년      |
| 100BASE-TX | 100Mbps   | UTP케이블 (Cat5 이상) | 100m             | 1995년      |
| 1000BASE-T | 1000Mbps  | UTP케이블 (Cat5 이상) | 100m             | 1999년      |
| 10GBASE-T  | 10Gbps    | UTP케이블 (Cat6 이상) | 100m             | 2006년      |

### 5장 : 네트워크 계층

- `네트워크 계층` : 서로 다른 네트워크의 목적지로 데이터 전송하기 위해 필요
- `라우터` : 해당 목적지까지 최적의 경로를 알려주는 장치
- `라우팅` : 목적지까지 어떤 경로로 데이터를 보낼지 결정하는 것. 라우팅 테이블을 가짐
- `IP 주소` : 어떤 네트워크의 어떤 컴퓨터인지 구분하는 주소
- `라우팅 테이블` : 경로 정보를 등록하고 관리하는 테이블
- 네트워크 계층에서는 캡슐화할 때 `IP 헤더`를 붙임.
- `IP 패킷` : IP 헤더를 추가해 만들어지는 것
- `IP 주소`에는 `IPv4`와 `IPv6`가 있다.
- 인터넷에 직접 연결되는 컴/라우터에는 `공인 IP 주소`, 회사나 가정의 랜 컴퓨터에는 `사설 IP 주소` 할당
- `DHCP` : 라우터에서 IP 주소를 자동으로 할당해주는 기능
- 옥텟 : 8비트 단위를 부르는 단위
- IP 주소는 `네트워크 ID(어느 네트워크)`와 `호스트 ID(어느 컴퓨터)`로 결정된다.

| 클래스 이름 | 내용             |
| ----------- | ---------------- |
| A 클래스    | 대규모 네트워크  |
| B 클래스    | 중형 네트워크    |
| C 클래스    | 소규모 네트워크  |
| D 클래스    | 멀티캐스트 주소  |
| E 클래스    | 연구 및 특수용도 |

- `A 클래스` : 앞 8비트가 네트워크 ID, 뒤 24비트가 호스트 ID (1옥텟 0~127, 호스트 수 1677만)
- `B 클래스` : 앞 16비트가 네트워크 ID, 뒤 16비트가 호스트 ID (1옥텟 128~191, 호스트 수 6만)
- `C 클래스` : 앞 24비트가 네트워크 ID, 뒤 8비트가 호스트 ID (1옥텟 192~223, 호스트 수 254)

| 종류     | 사설 IP 주소의 범위         |
| -------- | --------------------------- |
| A 클래스 | 10.0.0.0~10.255.255.255     |
| B 클래스 | 172.16.0.0~172.31.0.0       |
| C 클래스 | 192.168.0.0~192.168.255.255 |

- `네트워크 주소` : 호스트 ID가 0인 주소 (작은 네트워크의 대표 주소)
- `브로드캐스트 주소` : 호스트 ID가 11111111(255)인 주소 (모두에게 한번에 데이터 전송)
- `서브넷팅` : 네트워크를 분할하는 것. 대규모 네트워크 등을 작은 네트워크로 분할해 브로드캐스팅 시 혼잡해지는 것을 막음
- `서브넷` : 서브넷팅으로 분할된 네트워크
- 기존의 IP 구조가 `네트워크 ID` + `서브넷 ID` + `호스트 ID`의 구조로 나뉨
- `서브넷 마스크` : 서브넷팅 했을 때 네트워크 ID 및 호스트 ID 중 어디가 어딘지 식별하기 위한 값 (없으면 구분 어려움)
- `프리픽스 표기법` : 서브넷 마스크를 슬래스(/비트수)로 나타낸 것. (ex. 255.255.255.0은 /24가 됨) (네트워크 주소가 어디까지인지 정함)
<hr>

- 라우터를 이용해 네트워크를 분리할 수 있다. (스위치, 허브는 분리 불가능)
- `기본 게이트웨이` : 설정한 네트워크의 출입구 (기본적으로 1)
- 라우팅 테이블에 자동 등록/수동 등록 가능 (소규모 네트워크에서 수동 등록이 적합)
- `라우팅 프로토콜` : (라우터 간에) 라우팅 정보를 교환하기 위한 프로토콜 (ex. RIP, OSPF, BGF)

### 6장 : 전송 계층

- `전송 계층` : 목적지에 신뢰할 수 있는 데이터를 전달하기 위해 필요
- 오류 점검 기능, 전송된 데이터의 목적지가 어떤 애플리케이션인지 식별하는 기능
- `TCP` : 신뢰성, 연결형 프로토콜 (연결 확립, 재전송 제어, 윈도우 제어)
- `UDP` : 효율성, 비연결형 프로토콜
- TCP로 전송할 때 붙이는 헤더를 `TCP 헤더`, 이 TCP 헤더가 붙은 데이터를 `세그먼트`라고 한다.
- 통신하려면 `연결`이라는 가상의 독점 통신로를 확보해야 함
- TCP의 헤더 부분에는 `URG`, `ACK`, `PSH`, `RST`, `SYN`, `FIN` 등의 코드 비트가 있다.
- `SYN`은 연결 요청, `ACK`는 확인 응답, `FIN`은 연결 종료
- `3-way Handshake` : 연결을 확립하기 위해 패킷을 교환하는 과정 (SYN, SYN+ACK, ACK)
- `4-way Handshake` : 연결을 종료할 때 패킷 교환 과정 (FIN, ACK, FIN, ACK)
- TCP 헤더에서 실제 데이터를 주고받을 때는 `일련번호`와 `확인 응답 번호`를 사용
- `일련 번호` : 송신 측의 데이터가 몇 번째 데이터인지 알려줌 (패킷 순서)
- `확인 응답 번호` : 수신 측이 몇 번째 데이터를 수신했는지 송신 측에 알려줌 (+다음 데이터 요청)
- `재전송 제어` : 일련번호와 확인 응답 번호를 확인해 데이터 손상/유실의 경우 재전송함
- `윈도우 크기` : 얼마나 많은 용량의 데이터를 버퍼에 저장할 수 있는지 크기 (3-way 핸드셰이크 때 판단)
<hr>

- 무슨 애플리케이션인지 판단하기 위해, TCP 헤더에서 `출발지 포트 번호` 및 `목적지 포트 번호` 기록
- `잘 알려진 포트(well-known ports)` : 0~1023번 포트 (1024는 사용X, 1025 이상은 랜덤 포트)

| 애플리케이션 | 포트 번호 |
| ------------ | --------- |
| SSH          | 22        |
| SMTP         | 25        |
| DNS          | 53        |
| HTTP         | 80        |
| POP3         | 110       |
| HTTPS        | 443       |

- 서버 측에서는 포트 번호를 정해둬야 하지만, 클라이언트 측은 정하지 않아도 된다.
- `UDP 헤더`가 붙은 데이터를 `UDP 데이터그램`이라고 한다.
- UDP를 사용하면 `브로드캐스트`가 가능하다. (데이터를 일괄로 보내는 것)

### 7장 : 응용 계층

- 클라이언트의 요청을 전달하기 위해 서버가 이해할 수 있는 메시지로 변환하고 전송 계층으로 전달하는 역할을 함.
- 클라 애플리케이션-서버 애플리케이션 통신하려면 `응용 계층 프로토콜`이 필요함

| 프로토콜 | 내용           |
| -------- | -------------- |
| HTTP     | 웹 사이트 접속 |
| DNS      | 이름 해석      |
| FTP      | 파일 전송      |
| SMTP     | 메일 송신      |
| POP3     | 메일 수신      |
